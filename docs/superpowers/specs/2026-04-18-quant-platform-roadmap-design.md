# 量化交易平台能力补齐 —— 路线图设计

**日期:** 2026-04-18
**状态:** Roadmap spec（非单一实现 spec；用于拆分与排序）
**项目:** InStock

## 背景

InStock 当前已具备以下能力：日频 OHLCV 抓取、技术指标计算、K 线形态识别、若干选股策略、简单的事件型"选股后 N 日胜率"统计（`rate_stats`）、基于同花顺客户端的自动打新脚本、Flask Web 展示。

从一名严肃量化交易者 / 操盘手的角度评估，系统在**数据工程、回测严谨度、组合风险管理、实盘执行质量**四个方向存在系统性缺口。本文件不直接给出实现方案，而是将缺口拆分为 4 个独立子项目，确定依赖顺序，作为后续 4 份 spec 的总索引。

## 子项目拆分与依赖

```
[1. 数据 & 因子工程] ──→ [2. 回测引擎] ──→ [3. 风险 & 组合] ──→ [4. 实盘执行]
         │                                        ↑
         └────────────────────────────────────────┘
         (组合优化需要因子暴露 / 行业暴露数据)
```

依赖理由：
- 爬虫层当前是整个系统最脆弱的一环。无干净数据则回测结论失真，下游所有子系统都在沙上建塔。
- 风控参数（仓位上限、回撤阈值、止损点）应由回测的回撤分布反推，而非拍脑袋，故风控必须在回测之后。
- 实盘自动交易在策略未经严肃回测和组合风控前属于高风险，放在最后。

每个子项目**独立**进入 brainstorm → spec → plan → implementation 循环。完成一个再进入下一个。

---

## 子项目 1 — 数据 & 因子工程

**目标：** 把爬虫层从"能跑"升级到"可信"，并引入因子库框架。

**核心缺口：**
- 爬虫稳定性：统一超时 / 重试 / cookie / 代理的失败降级（当前 `singleton_proxy.py` 在 `proxy.txt` 缺失时 `self.data` 未定义，级联打挂所有爬虫）。
- 分钟级数据接入（至少 1min），为日内策略与滑点建模提供基础。
- 复权处理统一化：现在前/后复权逻辑散落各模块。
- 一致性校验：股票代码变更、ST 摘帽、退市、新股首日、停牌。
- **PIT（point-in-time）财务数据**：当前所有财务字段都是"最新值"，回测会严重前视偏差。需要按公告日而非报告期索引。
- 因子库框架：统一的因子定义接口（signature + 元数据）、横截面处理（去极值 / 行业市值中性化 / 标准化）、因子合成。
- 因子评估：IC、Rank IC、IC_IR、分层回测、换手率、因子衰减曲线。
- 龙虎榜席位画像（数据已在，未利用）：席位累计胜率、席位共现网络、帮派识别。
- 北向资金明细（当前未抓）。
- 事件数据：大股东增减持、回购、业绩预告、限售解禁。

**范围外（由子项目 2/3 承担）：** 回测、组合构建。

---

## 子项目 2 — 回测引擎

**目标：** 用事件驱动的组合级回测替换当前的"选股后胜率"事件研究。

**核心缺口：**
- 事件驱动架构（on_bar 逐条推进），而非 vectorized 胜率统计。
- 组合层回测（多票、资金曲线、仓位演化）。
- 真实交易成本：印花税 + 过户费 + 佣金 + 按成交额 / 波动率建模的滑点。
- A 股特殊规则：T+1、涨跌停 / 一字板无法成交、停牌停更。
- 除权除息的现金与再投资处理。
- Walk-Forward、样本内 / 外分离。
- 参数网格搜索 + 过拟合检测（deflated Sharpe）。
- 标准绩效指标：Sharpe / Sortino / Calmar / 最大回撤 / 回撤持续时间 / 胜率 / 盈亏比 / 年化换手。

**依赖：** 子项目 1 的干净数据 + PIT 财务。

---

## 子项目 3 — 风险管理 & 组合优化

**目标：** 引入"组合"这个一等概念，把多个策略的信号汇总为一张可风控的持仓表。

**核心缺口：**
- 组合状态：当前持仓、成本、浮动盈亏、可用资金。
- 事前风控：单票上限、行业上限、主题上限、总杠杆。
- 事中风控：动态止损（ATR / 移动止损 / 时间止损）、组合最大回撤熔断。
- 仓位分配方法：等权 / 波动率倒数 / 风险平价 / 凯利。
- 相关性与行业矩阵，避免多策略伪分散（"10 个策略全在买新能源"）。
- 策略表现监控：单策略 rolling Sharpe，3 个月内衰减则自动降权或停用。

**依赖：** 子项目 2 的回测引擎（用于风控参数标定）；子项目 1 的行业 / 因子暴露数据。

---

## 子项目 4 — 实盘执行

**目标：** 把单一同花顺打新脚本升级为可审计、可风控的多券商执行层。

**核心缺口：**
- 券商适配抽象层（当前与 `gf_client` 耦合过紧）。
- 模拟盘 / 纸面交易模式（切实盘前的强制中间层）。
- 订单拆单：TWAP / VWAP，避免大单冲击。
- 成交回报与信号对账（应成交 vs 实际成交，回推实际滑点）。
- 实时风控：单日亏损熔断、异常下单防护。
- 交易审计：每笔订单可追溯至（策略 ID, 信号时间戳, 当时因子值, 当时组合状态）。

**依赖：** 子项目 3 的组合风控输出。

## 非目标（本路线图明确不做）

- 机器学习模型工厂（不反对，但不是平台义务；留给策略研究层）。
- 期货 / 期权 / 港美股扩展（先把 A 股做扎实）。
- 可视化 BI 扩展（Flask Web 当前足够）。
